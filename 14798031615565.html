San Francisco 字体笔记

2016/11/22 posted in  字体 comments

这篇文章的内容总结于字体播客节目 Type is Beautiful 的「苹果字体极简史-圣弗朗西斯科」，里面除了 San Francisco 的部分还讲到了很多字体设计上有趣的知识，正好最近在看小林章的《西文字体》，对字体相关的内容就比较感兴趣了，所以就把这些东西总结在博客里，希望能够活学活用吧。

目录

苹果前年在 WWDC 2015 上介绍了一款全新的系统字体 San Francisco ，这款字体被广泛应用在苹果系统界面，硬件产品，以及市场营销上。现在看来，这是一款成熟度非常高的字体，如此广的字符集和适用型应该是无法在一两年内能够完成的，可以说苹果公司在系统界面字体上面布了一个很长的棋，花费了很多精力做了一套全新的字体出来。

下面会围绕着这款字体的「特殊之处」「设计」和「应用」三个方面来做一个简单的介绍。

1. 特殊之处

San Francisco 第一次出现是在苹果手表上，当时刚发布的时候其实给人感觉有点像德国的 DIN 字体，DIN 是德国工业标准字体，非常适合做导向字体，字母看齐来比较方🌝，比如字母 O 就挺像跑道的。

两款字体

严格来讲，San Francisco 共包含了两款字体：为 iOS，macOS，tvOS 设计的 SF，以及为 Watch OS 设计的 SF Compact 。两款字体从字形上来讲相差还挺大的，SF Compact 比较像 DIN，而 SF 更接近 Helvetica 。但可能是苹果为了品牌概念的营销，所以被都叫作 San Francisco 了。

单个字母来看， SF Compact 要显得更方一些（垂直线弧度比较小），这样差异化的设计其实是为了解决字体在不同载体上的易认性问题，因为 SF Compact 字体是应用在手表上的，手表上字体较小，使用场景通常是瞥一眼来获取信息（官方说要为 Glance 而设计），这种需求其实就有点类似与公路上的导向字体，对易认性都有很高的要求（公路导向字体需要尽量保证远距离大雾天气的字体的可识别性）。如果两种字体两个字号一样，x-height 一样，一个方的一个圆的，为了保证曲线的感觉，圆的字在横向扩展会宽一些，排的就比较少；而方的字在竖直的部分会使得各字的领域保持的较好一些，不会出现曲线靠近后出现黏在一起的感觉，能够显示更多字母，保证有限空间内显示更多信息。这或许也是被称为 Compact 的缘故吧。

导向字体和正文字体不同的是，导向字体通常是为保证每个字母都能看清而设计的。而正文字体通常是则是为单词的形状（Word Shape）而设计的，因为在读文章的时候，看的通常是一个一个单词，而不是一个一个字母。其实汉字有时候也这样，不是经常有人提笔忘字吗，比如我大概记得这个字怎么写，但忘记了具体的笔画，这其实就是记得字的形状而不记得其结构。另外微软在做 Clean Type 的时候也做过阅读时眼睛聚焦部分的研究。

为什么要区分 Text 和 Display

这两种字体同时又分别分为 Text 和 Display 两种（可以翻译为是正文字和标题字，有人将 Display 译为屏显，但这样其实这太好理解），在数位字体上很少看到有这样分类的，但这也不算是苹果首创。Adobe 早期提过类似的概念—Optical Size（视觉字号），并在2000年的时候推出了具有 Optical Size 特性的 Minion 字体。但后来，也许是不太易用的缘故，使用这款字体的人并不是很多。

套用中文字体来讲的话，这其实有点类似于中文的「书宋」和「标宋」，顾名思义书宋就是给正文使用的，标宋就是给标题使用的。由于使用场景不同，字形也有所差异，比如标宋的横竖粗度对比强烈，横是比较细的，会有一定的装饰感，显得比较秀气，但是如果用标宋来排正文字的话，细横可能就看不见了，相反如果书宋如果放大做标题字的话，就会显得笔画比较粗大，笨重，不够优雅了。

所以，区分标题和正文的字的字体设计是自古就有的。不过在这之前没有被普及是因为这种分类方式对大多数人来说并不了解，而且也不易用。但是 San Francisco 这次却采用了这种方式而且是应用在 UI 字体上，其实是让人有的吃惊的，至于 San Francisco 是怎么解决易用的问题的呢？在后面的应用-这里放个连接段落会有所介绍。

Text 和 Display 的差异

字重上 Text 的字重有6级，Display 的字重有9级。Display 多的3个字重是在更细的区域要多2个，更粗的区域多1个。因为特别细或者特别粗的字对字号是有一定要求的，特别细的字体排的过小可能就断线了，特别粗的字排的过小就会糊成一团，中间的空白可能就看不清了，所以也只有大标题可以将细字体和粗字体显示清楚。

同时，苹果为了保证在不同字号下字体的易认性，Display 和 Text 的造型设计也是有微妙变化的。比如 Display 的字母开口区域更大一些，以遍清晰的显示标题。另外小写字母 r ，Display 伸出来的耳朵比较短，text 的就要长一些。

2. 设计

无衬线字体

毫无疑问 San Francisco 属于无衬线字体，相较与著名的 Helvetica ，San Francisco 的边角更为锐利一些，更适合作为界面字体。

比例字体

比例字体，指的是字符宽度不尽相同的电脑字体，相反如果所有字符宽度一致被称为等宽字体。等宽字体的出现最初是因为以前的打印机没有办法很好的排版，所以每个字符用的是一样的宽度；不过也因为每个字符宽度一致，方便对齐，能非常清楚的让你看到字母数量，所以程序员也比较偏爱等宽字体。

San Francisco 也是一款比例字体。

正文文字的拉丁字符为了方便阅读则尽量不要等宽，而且还有做 Kerning，让某些字符有所交集，比如 ﬂ 等。中文的话由于是方块字，字符大小都是一样的，所以就没有区分等宽或非等宽了。

等高数字

  • 等高数字（ Lining Figures 或称作线性数字或大写数字）：顾名思义就是每个阿拉伯数字高度一样，San Francisco 的等高数字就是和其大写字母一样高，等高数字通常被认为更适合搭配大写字母或在表单中使用。
  • 非等高数字（ Text Figures 或称作文字数字或小写数字）：源于老式的数字写法，是一种模仿拉丁字母高低错落而设计的数字，比较错落有致，与英文字母排列起来比较和谐，不会特别突出，比较适合正文排版，比如 Georgia 字体

OpenType 特性

OpenType 是一种可以支持丰富排版效果的字体格式，San Francisco 支持上下文替代字，等宽数字，分数字等具体效果如下图

丰富的字体家族成员

支持东欧文字-波兰文；北欧文字-冰岛语，挪威语，希腊文；支持西里尔字母（俄文、乌克兰文、塞尔维亚文等所使用的字母）；支持国际音标字符，越南文字母和各种附加符号和声调。几乎可以说 San Francisco 把所有泛指的西文的字符都包含进来了。覆盖了极广的国家和语言。

3. 应用

为了方便开发者更好的使用字体，苹果系统会随着字体尺寸的变化自动调用 Text 或 Display 的字体，字体尺寸大于 20pt 的会使用 Display 这一套字体，相反，小于 20pt 就会使用 Text 这一套。而且，为了提高易认性，在不同尺寸下，字形，字偶间距（kerning）和字符间距（tracking）也会有细微变化。

另外，为了让开发者更方便的调节字体参数，苹果还另外设计了一套 Dynamic Type 机制，在 Dynamic Type ，苹果一开始就订好了他们认为最好的一套一套的 Style，可以通过 API 直接调用。Style 里面包含了字体家族，字重，行高，字符间距等，当开发者自动选好字号后，系统就会自动调用相应的风格的 Display 或 Text 字体。（但是目前这套机制对中文似乎支持的不太好，国内应用也鲜有使用。）

总结

在传统的手工物理造字的时代，字体有很多很好的特性，但是因为技术过渡导致这些优良特性都没有被延续过来。而苹果这一次充分发挥了数位字体的特点，将 Optical Size 再一次重提出来，同时为开发者提供了 Dynamic Type 和丰富的 API，让过去的造字工艺的优点得以重新发挥。

后记

最近听IT工论讲到了一个民主化的概念，和字体排印的发展其实有很多相通之处：政治制度的民主，它带来的坏处是效率是低的（和专制相比较），但好处是可以促进社会阶层的流动，另外就是可以各种人都可以用到各种东西，但是同时也有可能走向民粹化。设计角度来讲的话，以前就是少数人（排字工）来做的工作，由于人少，各种也有明确的规则，一旦权利下放之后，普罗大众有了这个权利，普罗大众是没有必要学习这种规则，但是他又有了权利，这就会导致很多业娱的排版的出现。所以像苹果这种提供「工具箱」的公司，能够为产品设计者提供很多预制模板就很有必要了。
