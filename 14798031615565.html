San Francisco 字体笔记

2016/11/22 posted in  播客笔记  

这篇文章的内容总结于字体播客节目 Type is Beautiful 的「苹果字体极简史-圣弗朗西斯科」，里面除了 San Francisco 的部分还讲到了很多字体设计上有趣的知识，正好最近在看小林章的《西文字体》，对字体相关的内容就比较感兴趣了，所以就把这些东西总结在博客里，希望能够活学活用吧。

苹果去年在 WWDC 2015 上介绍了一款全新的系统字体 San Francisco ，这款字体被广泛应用在苹果系统界面，硬件产品，以及市场营销上。现在看来，这是一款成熟度非常高的字体，如此广的字符集和适用型应该是无法在一两年内能够完成的，可以推断苹果公司在系统界面字体上面布了一个很长的棋，花费了很多精力做了一套全新的字体出来。

San Francisco 第一次出现是在苹果手表上，当时刚发布的时候其实给人感觉有点像德国的 DIN 字体，DIN 是德国工业标准字体，非常适合做导向字体，字母看齐来比较方🌝，比如字母 O 就挺像跑道的。

严格来讲，San Francisco 共包含了两款字体：为 iOS，macOS，tvOS 设计的 SF，以及为 Watch OS 设计的 SF Compact 。两款字体从字形上来讲相差还挺大的，SF Compact 比较像 DIN，而 SF 更接近 Helvetica 。但可能苹果为了品牌概念的考虑，所以被都叫作 San Francisco 了。

同时这两种字体又分别分为 Text 和 Display 两种（可以翻译为是正文字和标题字，有人将 Display 译为屏显，但这样其实这太好理解），在数位字体上很少看到有这样分类的，但这也不算是苹果首创。Adobe 早期提过类似的概念—Optical Size（视觉字号），并在千禧年的时候推出了具有 Optical Size 特性的 Minion 字体。但后来，也许是不太易用的缘由，使用这款字体的人并不是很多。

套用中文字体来讲的话，其实有点类似于中文的「书宋」和「标宋」，顾名思义书宋就是给正文使用的，标宋就是给标题使用的。由于使用场景不同，字形也有所差异，比如标宋的横竖粗度对比强烈，横是比较细的，会有一定的装饰感，显得比较秀气，但是如果用标宋来排正文字的话，细横可能就看不见了，相反如果书宋如果放大做标题字的话，就会显得笔画比较粗大，笨重，不够优雅了。

所以，总得来说，区分标题和正文的字的字体设计是自古就有的。不过在这之前没有被普及是因为这种分类方式对大多数人来说并不用，而且也不易用。但是 San Francisco 这次却采用了这种方式而且是应用在 UI 字体上，其实是让人有的吃惊的，那么 San Francisco 是怎么解决易用的问题的呢？

首先先来了解下 Text 和 Display 。Text 的字重有6级，Display 的字重有9级。Display 多的3个字重是在更细的区域要多2个，更粗的区域多1个。因为特别细或者特别粗的字对字号是有一定要求的，特别细的字体排的过小可能就断线了，特别粗的字排的过小就会糊成一团，中间的空白可能就看不清了，所以也只有大标题可以将细字体和粗字体显示清楚。同时，苹果为了保证在不同字号下字体的易认性，Display 和 Text 的造型设计也是有微妙变化的。比如 Display 的字母开口区域更大一些，以遍清晰的显示标题。另外小写字母 r ，Display 伸出来的耳朵比较短，text 的就要长一些。在 SF 和 SF Compact 中各有一套这种，单纯看字重的话就包含了差不多 (6+9)×2=30 套了，这还没有算 Text 里面意大利斜体的部分，所以 San Francisco 真的是一个很大的家族啊。

另外再了解下 SF 和 SF Compact 字体造型的差异。单个字母来看， SF Compact 要显得更方一些（垂直线弧度比较小），之所以有这样差异化的设计其实是一个易认性的问题，因为 SF Compact 字体是应用在手表上的，手表上字体较小，使用场景通常是瞥一眼来获取信息（官方说要为 Glance 而设计），这种需求其实就有点类似与公路上的导向字体，对易认性都有很高的要求（公路导向字体需要尽量保证远距离大雾天气的字体的易认性）。

导向字体和正文字体不同的是，导向字体通常是为保证每个字母都能看清而设计的。而正文字体通常是则是为单词的形状（Word Shape）而设计的，因为在读文章的时候，看的通常是一个一个单词，而不是一个一个字母。其实汉字有时候也这样，不是经常有人提笔忘字吗，比如我大概记得这个字怎么写，但忘记了具体的笔画，这其实就是记得字的形状而不记得其结构。另外蔚然在做 Clean Type 的时候也做过阅读时眼睛聚焦部分的研究。

如果两个字号一样，x-height 一样，一个方的一个圆的，为了保证曲线的感觉，圆的字在横向扩展会宽一些，排的就比较少；而方的字在竖直的部分会使得各字的领域保持的较好一些，不会出现曲线靠近后出现黏在一起的感觉，能够显示更多字母，保证有限空间内显示更多信息。这其实也是 Compact 名字的来源。
